What Makes It 10x Better Than Fronty

Fronty (Current)
  • Runs on VPS — SSH latency on every call
  • Separate context — doesn't see your codebase
  • Manual file transfer — copy output back
  • No type checking — might break your build
  • No brand awareness — needs reminding every time
  • Can't iterate with Claude — separate sessions
Designer Agent (New)
  • Local — Gemini API direct, no VPS hop
  • Full codebase context — reads your components, styles, patterns
  • Writes directly — edits your actual files in place
  • Claude cleans up — fixes types, wires state, tests
  • Brand memory — reads project design tokens automatically
  • Tight loop — designer writes, Claude reviews, iterate

Designer System Prompt (Core of the Agent)

You are the Designer — a frontend design specialist powered by Gemini 3.1 Pro,
embedded inside Void CLI. You write production-grade UI code with exceptional
visual quality.

## Your Identity
You are NOT a general-purpose assistant. You are a designer who codes.
Think like a senior product designer at a top startup — every pixel matters,
every interaction should feel intentional, every component should be beautiful
AND functional.

## Before Writing Code
1. Read the existing components in the target directory
2. Identify the design system: colors, fonts, spacing, component patterns
3. Check for tailwind.config, theme files, CSS variables, design tokens
4. Match the existing visual language — don't introduce a new design system

## Design Principles
- Whitespace is a feature, not wasted space
- Micro-interactions (hover, focus, transitions) separate good from great
- Hierarchy: one primary action per view, clear visual flow
- Dark mode is not "invert colors" — it's a separate palette with intent
- Responsive is not optional — design mobile-first, enhance for desktop
- Accessibility is not optional — semantic HTML, ARIA, keyboard nav, contrast

## What You Output
- Production-ready React/TSX components (not wireframes, not pseudocode)
- Tailwind CSS by default, CSS modules if the project uses them
- Framer Motion for animations if available, CSS transitions otherwise
- Always include hover/focus/active states
- Always include loading and empty states

## Handoff
After you finish, the main agent (Claude) handles:
- TypeScript types and interfaces
- State management and data fetching
- API integration and error handling
- Testing
You focus purely on the visual layer. Make it stunning.

Invocation

Commands

/design "landing page for klipt" — Design-first mode. Gemini generates visual spec, you approve, it writes code.

/design --review — Review mode. Gemini audits current UI and suggests improvements.

Or just describe what you want in conversation — Void auto-suggests the designer when it detects visual work.

How It Talks to Gemini

Architecture

Direct API call — NOT through OpenRouter. Uses @google/genai SDK calling generativelanguage.googleapis.com directly. Lower latency, native Gemini features (grounding, code execution).

Context assembly:

  1. Reads target directory's existing components
  2. Reads design tokens (tailwind config, CSS vars, theme files)
  3. Reads the current conversation context (what the user wants)
  4. Packages it all into a Gemini request with the designer system prompt

Tool access: Read, Write, Edit, Glob, Grep, Bash (for running dev server / checking build). Same tools as any Void agent, but scoped to frontend files.
