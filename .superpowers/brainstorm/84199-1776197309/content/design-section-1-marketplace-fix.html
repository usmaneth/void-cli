Design Section 1: Fix Plugin Marketplace Access

One-line fix unlocks all 33 official plugins for Void

The Problem

Void tries to install from anthropics/void-plugins-official — a repo that doesn't exist. Both the GCS CDN path and the git fallback point to the wrong name. Claude Code uses anthropics/claude-plugins-official which has all 33 plugins.

The Fix (3 files, ~6 lines)

src/utils/plugins/officialMarketplace.ts
- repo: 'anthropics/void-plugins-official'
+ repo: 'anthropics/claude-plugins-official'

- OFFICIAL_MARKETPLACE_NAME = 'void-plugins-official'
+ OFFICIAL_MARKETPLACE_NAME = 'claude-plugins-official'
src/utils/plugins/officialMarketplaceGcs.ts
- GCS_BASE = '...plugins/void-plugins-official'
+ GCS_BASE = '...plugins/claude-plugins-official'

- ARC_PREFIX = 'marketplaces/void-plugins-official/'
+ ARC_PREFIX = 'marketplaces/claude-plugins-official/'

Result

After this fix, /plugin in Void shows the same 33 plugins as Claude Code: superpowers, figma, slack, telegram, frontend-design, code-review, pr-review-toolkit, LSP integrations, and everything else.
