Design Section 3c: Smart Mode Triggers

Void dynamically suggests deliberation and swarm — like how superpowers auto-invokes skills

The Pattern

Today, superpowers skills fire automatically based on context — brainstorming triggers before creative work, TDD triggers before implementation. Deliberation and Swarm should work the same way: the main Void agent recognizes when these modes would help and suggests them inline.

What It Looks Like

Example 1: Auto-suggest Deliberation
you › should we use websockets or SSE for the real-time updates?
 
void › This is a meaningful architecture decision with real tradeoffs.
       I can answer this myself, or we can deliberate on it —
       I'd pull in GLM 5.1 and Gemini to stress-test the choice
       before we commit. Want to open the room?
 
you › yeah do it

Example 2: Auto-suggest Swarm
you › build a settings page with a form, API endpoint to save
       preferences, and database migration for the new table
 
void › This has clean frontend/backend/data separation — perfect
       for swarm mode. I'd split it:
       • Gemini → settings form UI + responsive layout
       • GPT-5.4 → API route + validation + DB migration
       • I'll wire them together after
       Should I swarm it or handle it myself?
 
you › swarm it

Trigger Heuristics

Deliberate
  • Architecture decisions
  • "Should we X or Y?"
  • Tradeoff discussions
  • Design pattern selection
  • "What's the best way to..."
  • User explicitly stuck
Swarm
  • Multi-layer features (FE+BE+DB)
  • Large feature with clear boundaries
  • "Build X with Y and Z"
  • Full-stack task description
  • 3+ independent components
  • User says "full feature"
Designer
  • "Make it look better"
  • "Redesign this"
  • "10x the UI"
  • Editing .tsx with layout work
  • "Beautiful", "polished", "modern"
  • Visual mockup requests

Implementation: Registered as Skills

Each mode registers as a built-in skill with trigger conditions — same mechanism superpowers uses. The main agent's system prompt includes awareness of all three modes and when to suggest them.

src/skills/bundled/deliberate.ts
name: 'deliberate'
description: 'Multi-model deliberation for hard decisions'
whenToUse: 'Architecture decisions, tradeoff discussions,
  design pattern selection, or when user is stuck on a
  choice between approaches'
aliases: ['debate', 'discuss']
src/skills/bundled/swarm.ts
name: 'swarm'
description: 'Multi-model parallel implementation'
whenToUse: 'Multi-layer features with clear frontend/backend/
  data separation, large features with 3+ independent
  components, or full-stack tasks'
aliases: ['crew', 'team']

Key: Void always suggests the mode, never auto-launches. You stay in control — "yeah do it" or "nah I'll handle it" and the agent continues normally.
